# IT3040 Assignment 1 - Transliteration Accuracy Testing

## Project Overview

This project was created for **IT3040 - ITPM Assignment 1, Option 1: Transliteration Accuracy Testing**.

The purpose of this project is to automate test cases for the Chat Sinhala transliteration function available at:

```text
https://www.pixelssuite.com/chat-translator
```

The test cases evaluate how accurately the system converts chat-style Singlish input into Sinhala output.

## Scope of Testing

This project focuses only on the **Chat Sinhala transliteration function**.

The following areas are outside the scope of this project:

- Standard Sinhala transliteration
- Backend API testing
- Performance testing
- Scalability testing
- Security testing

## Prerequisites

Before running the automation script, make sure the following are installed on your computer:

- Python 3.11 or Python 3.12
- Google Chrome
- Playwright
- openpyxl

## Installation and Setup Instructions

### Step 1: Download or Clone the Repository

Download or clone this repository to your local machine.

Example location:

```text
C:\Users\Lenovo\Desktop\ITPM_Assignment_v2
```

### Step 2: Open the Project Folder

Open the project folder in VS Code or open PowerShell/Command Prompt in the project directory.

If the automation files are inside the `test_automation` folder, navigate to it:

```powershell
cd "C:\Users\Lenovo\Desktop\ITPM_Assignment_v2\test_automation"
```

### Step 3: Create a Virtual Environment

Run the following command:

```powershell
python -m venv .venv
```

### Step 4: Activate the Virtual Environment

For PowerShell, run:

```powershell
.\.venv\Scripts\Activate.ps1
```

If PowerShell blocks the activation command, run:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned
```

Then activate the virtual environment again:

```powershell
.\.venv\Scripts\Activate.ps1
```

After activation, the terminal should show:

```text
(.venv)
```

### Step 5: Upgrade pip

Run:

```powershell
python -m pip install -U pip
```

### Step 6: Install Required Dependencies

Run:

```powershell
python -m pip install playwright openpyxl
```

### Step 7: Install Playwright Browsers

Run:

```powershell
python -m playwright install
```

## Preparing the Excel File

Before running the automation script, open the Excel file:

```text
Assignment 1 - Test cases.xlsx
```

Fill only the following columns manually:

```text
TC ID
Input length type
Input
Expected output
```

Do not manually fill the following columns before running the automation script:

```text
Actual output
Status
```

The automation script will automatically update the `Actual output` and `Status` columns.

## Test Case ID Format

Since this assignment focuses on negative test cases, all test case IDs are written using the following format:

```text
Neg_0001
Neg_0002
Neg_0003
...
Neg_0050
```

## Input Length Type

The input length type should be entered based on the length of the Singlish input:

```text
S = 30 characters or less
M = 31 to 299 characters
L = 300 to 450 characters
```

## Running the Automation Script

Before running the script, make sure the Excel file is closed.

Run the following command from the folder where `test_automation.py` is located:

```powershell
python test_automation.py --excel "Assignment 1 - Test cases.xlsx" --url "https://www.pixelssuite.com/chat-translator" --wait-ms 5000 --type-delay-ms 80 --slow-mo-ms 200 --save-every 1 --keep-open
```

## Checking the Results

After the script finishes running:

1. Open `Assignment 1 - Test cases.xlsx`.
2. Check the `Actual output` column.
3. Check the `Status` column.
4. Confirm whether the test cases are marked correctly.

A test case is marked as `Fail` when the actual output generated by the system does not match the expected Sinhala output.

## Final Excel Columns

The final Excel file should contain the following columns:

```text
TC ID
Input length type
Input
Expected output
Actual output
Status
Singlish input types covered
Evidence or rationale for the input type covered
```

The following two columns should be added and filled manually after checking the automation results:

```text
Singlish input types covered
Evidence or rationale for the input type covered
```
